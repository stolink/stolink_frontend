## SaaS UX 설계 원칙

---

## 1. 온보딩 (First Experience)

### 1.1 Time to Value 최소화

```
사용자가 가치를 느끼기까지 걸리는 시간을 최소화

❌ 나쁨: 회원가입 → 튜토리얼 10분 → 설정 → 드디어 사용
✅ 좋음: 회원가입 → 즉시 핵심 기능 체험 → 나중에 설정
```

### 1.2 점진적 공개 (Progressive Disclosure)

```
모든 기능을 한 번에 보여주지 않음

1단계: 핵심 기능만 (에디터, 저장)
2단계: 사용하면서 발견 (복선 태그, AI 분석)
3단계: 고급 기능 (내보내기, 협업)
```

### 1.3 빈 상태(Empty State) 활용

```
┌─────────────────────────────────┐
│                                 │
│     📚 아직 작품이 없어요        │
│                                 │
│   첫 작품을 만들어보세요!        │
│                                 │
│   [+ 새 작품 만들기]            │
│                                 │
│   또는 샘플 작품 둘러보기 →      │
│                                 │
└─────────────────────────────────┘
```

## 2. 인지 부하 감소

### 2.1 청킹 (Chunking)

```
정보를 의미 있는 그룹으로 분류

❌ 나쁨: 설정 50개가 한 페이지에
✅ 좋음:
   [계정] [에디터] [알림] [내보내기]
      ↓
   선택한 카테고리만 표시
```

### 2.2 일관된 패턴

```
같은 동작 = 같은 UI

저장: 항상 Ctrl+S, 항상 우상단
삭제: 항상 빨간색, 항상 확인 모달
생성: 항상 + 버튼, 항상 같은 위치
```

### 2.3 인지 vs 기억

```
❌ 기억하게 하지 말고
✅ 인지하게 하라

❌ "단축키: Ctrl+Shift+Alt+F"
✅ 버튼에 단축키 힌트 표시: [분석 (Ctrl+S)]
```

## 3. 피드백 & 상태

### 3.1 시스템 상태 가시성

```
사용자는 항상 현재 상태를 알아야 함

저장 상태:  ✓ 저장됨  |  ⏳ 저장 중...  |  ⚠️ 저장 안 됨
동기화:     🟢 연결됨  |  🔴 오프라인
작업 진행:  ████████░░ 80% AI 분석 중
```

### 3.2 즉각적 피드백

```
모든 액션에 0.1초 이내 반응

클릭 → 버튼 눌림 효과 (즉시)
저장 → "저장 중..." 표시 (즉시)
완료 → 토스트 알림 (완료 시)
```

### 3.3 에러 처리

```
❌ 나쁨: "오류가 발생했습니다"
✅ 좋음:
   ┌─────────────────────────────┐
   │ ⚠️ 저장에 실패했어요         │
   │                             │
   │ 인터넷 연결을 확인해주세요    │
   │                             │
   │ [다시 시도]  [로컬에 저장]   │
   └─────────────────────────────┘
```

## 4. 효율성

### 4.1 단축키 & 파워유저 지원

```
초보자: GUI로 모든 기능 접근 가능
파워유저: 단축키로 빠르게

Ctrl+S     저장
Ctrl+/     복선 태그
Ctrl+K     명령 팔레트 (모든 기능 검색)
```

### 4.2 명령 팔레트 (Command Palette)

```
Ctrl+K 또는 Cmd+K

┌─────────────────────────────────┐
│ 🔍 명령어 검색...               │
├─────────────────────────────────┤
│ 📄 새 챕터 만들기               │
│ 🔍 복선 검색                    │
│ 📊 AI 분석 실행                 │
│ ⚙️ 설정 열기                    │
└─────────────────────────────────┘
```

### 4.3 자동 저장 & 복구

```
사용자가 "저장"을 생각하지 않게

- 자동 저장 (1.5초 디바운스)
- 버전 히스토리
- 실수로 삭제해도 복구 가능
- 브라우저 꺼져도 드래프트 보존
```

## 5. 정보 계층 (Visual Hierarchy)

### 5.1 F-패턴 & Z-패턴

```
사용자 시선 흐름에 맞게 배치

F-패턴 (텍스트 많은 페이지):
████████████████
████████████
████████
████

Z-패턴 (랜딩 페이지):
로고 ─────────────→ CTA
         ↘
           콘텐츠
         ↙
가격 ←─────────────── 시작하기
```

### 5.2 시각적 무게

```
중요도에 따른 강조

[저장] ← Primary (강조색, 크게)
[취소] ← Secondary (회색, 작게)
삭제   ← Tertiary (텍스트만)
```

### 5.3 여백의 활용

```
관련 있는 것은 가깝게, 없는 것은 멀게

┌─────────────────────────────────┐
│ 작품 정보          ← 그룹 제목   │
│ ┌─────────────────────────┐    │
│ │ 제목: [        ]        │    │ ← 관련 항목
│ │ 장르: [드롭다운]        │    │    가깝게
│ └─────────────────────────┘    │
│                                 │ ← 여백으로 구분
│ 내보내기           ← 다른 그룹   │
│ ┌─────────────────────────┐    │
│ │ [PDF] [마크다운] [EPUB] │    │
│ └─────────────────────────┘    │
└─────────────────────────────────┘
```

## 6. 접근성 (Accessibility)

### 6.1 키보드 네비게이션

```
Tab으로 모든 요소 접근 가능
Enter로 실행
Esc로 취소/닫기
```

### 6.2 색상 대비 & 색맹 고려

```
❌ 색상만으로 구분: 🔴 실패  🟢 성공
✅ 아이콘+텍스트 병행: ❌ 실패  ✓ 성공
```

### 6.3 반응형

```
데스크톱 → 태블릿 → 모바일

3단 레이아웃 → 2단 → 1단 (사이드바 접힘)
```

---

## 7. 신뢰 & 안전

### 7.1 파괴적 동작 보호

```
삭제, 덮어쓰기 등 되돌릴 수 없는 동작

1단계: 확인 모달
2단계: 중요한 건 타이핑 확인 "작품명을 입력하세요"
3단계: 30일 휴지통 보관
```

### 7.2 데이터 보존 신뢰

```
"내 글이 안전하다"는 확신

- 실시간 저장 상태 표시
- 마지막 저장 시간 표시
- 버전 히스토리 접근 가능
- 백업/내보내기 쉽게
```

### 7.3 프라이버시 존중

```
- 최소한의 정보만 수집
- 데이터 사용 투명하게 공개
- 계정 삭제 쉽게
- 내 데이터 내보내기 가능
```

│           │                             │                   │
